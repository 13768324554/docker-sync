docker-sync

Run your application at full speed while syncing your code for development, finally empowering you to utilize Docker for development under OSX

Developing with Docker under OSX is a huge pain, since sharing code into Docker containers make the code-execution about 50-60 times slower (depends on the solution). Testing and working with a lot of the alternatives made me pick the best of those, and combine this in one single tool: docker-sync

gem install docker-sync

Docker-sync is:

  • able to run on all of those docker-machines and also on Docker for mac
  • using either native_osx or unison/rsync to sync. The container performance is not influenced at all, see performance
  • Now a lot mor efficient in its way to watch for file changes - even for 50k+ files it will not eat up your CPU
  • supporting either one-way sync ( rsync ) or two way sync ( unison ), see strategies
  • has no longer dependencies on OSX when using native_osx
  • now supports background more to get out of your way
  • supporting user-remapping on sync to avoid permission problems on the container
  • supporting docker-compose to start sync and stack with one single command
  • using overlays to keep your production docker-compose.yml untouched and portable

Besides performance being the first priority for docker-sync, the second is, not forcing you into using a specific Docker solution. Use docker-for-mac, Docker toolbox, VirtualBox, VMware Fusion or Parallels, xhyve or whatever!
