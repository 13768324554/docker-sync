docker-sync

Run your application at full speed while syncing your code for development, finally empowering you to utilize docker for development under OSX/Windows/Linux*

Developing with docker under OSX/Windows is a huge pain, since sharing code into docker containers make the code-execution about 60 times slower (depends on the solution). Testing and working with a lot of the alternatives made us pick the best of those, and combine this in one single tool: docker-sync

gem install docker-sync

..detailed installation

Docker-sync is:

  • able to run on all of those docker-machines and also on docker for mac and 'Windows Ubuntu Shell'+docker
  • uses either native_osx>,unison or rsync as possible implementations. The container performance is not influenced at all, see performance
  • now a lot mor efficient in its way to watch for file changes - even for 50k+ files it will not eat up your CPU
  • supports either native_osx with file system events, unison two way and rsync one-way, see strategies
  • has no longer dependencies on OSX when using native_osx
  • now supports background more to get out of your way
  • supports user-remapping on sync to avoid permission problems on the container
  • support for docker-compose to start sync and stack with one single command
  • using overlays to keep your production docker-compose.yml untouched an portable
  • supports Linux* to use the same toolchain across all platforms, but maps on a native mount in linux (no sync)

Besides performance being the first priority for docker-sync, the second is, not forcing you into using a specific docker solution. Use docker-for-mac, docker toolbox, VirtualBox, VMware Fusion or Parallels, xhyve or whatever!
